--- layout: post title: "What is good design" description: "" category: tech tags: [design] --- {% include JB/setup %} A while ago I was asked "what is good design?" and I don't think I gave a great answer. There's plenty of existing literature on this subject but I thought I would answer it from the perspective of a software engineer.
Good design is functional
Users utilize software to accomplish tasks such as emailing, searching, listening to music, and so on. If a user interface doesn't help the user because it's buggy or presents obstacles then it's failed. Users can actually be very forgiving of "poorly designed" software as long as it addresses their goals:

CraigsList "Poorly designed" but very functional
Plenty of Fish Ads galore but still the most popular dating site
So what can I do as an engineer? It's easier said than done but I strive to create functional, bug-free code that helps users; this involves continuous testing, user/customer interviews, and well written software.
Good design is simple
User interfaces often create extra work for users by displaying too much irrelevant information. Here are some quotes that illustrate the importance of simplicity:

"Simplicity is the ultimate sophistication." Steve Jobs, quoting Leonardo da Vinci
"If I had more time, I would have written a shorter letter" Mark Twain
As a matter of fact, I often go through posts trying to remove as many words as possible.
Great design elicits emotion
As an engineer I hope to improve users' lives and create an impact with an ultimate goal of eliciting feelings which may include happiness, delight, surprise, and so on.
